# Graylog 로그 수집 플랫폼 구축 작업일지

- **작업일:** 2026-05-05
- **작업자:** 심제범
- **대상 서버:** vm-server-01 (172.18.100.20) — Graylog 호스트
- **로그 수집 대상:** pdkcld1, vm-server-01, gpu-server (syslog), 주요 Docker 컨테이너 (GELF)

---

## 1. 배경

Prometheus + Grafana로 **메트릭**은 수집 중이었으나 **로그** 수집 체계가 없었음.
장애 발생 시 각 서버에 SSH로 직접 접속하여 로그를 확인해야 했고, 보안 이벤트(SSH 무차별 대입 등) 감지 수단이 없었음.

MongoDB + OpenSearch는 이미 운영 중(7주 이상)이었으나 Graylog 컨테이너 자체가 주석 처리된 상태였음 → 활성화로 즉시 구축 완료.


## 3. 구성 내역

### 3-1. Graylog 컨테이너 활성화

**파일:** `/data/docker/docker-compose.yml` (vm-server-01)

```yaml
graylog:
  image: graylog/graylog:5.2
  container_name: graylog
  restart: always
  depends_on:
    - mongo
    - opensearch
  environment:
    GRAYLOG_PASSWORD_SECRET: "<96자 랜덤 시크릿>"
    GRAYLOG_ROOT_PASSWORD_SHA2: "<SHA2 해시>"
    GRAYLOG_HTTP_EXTERNAL_URI: "http://172.18.100.20:9000/"
    GRAYLOG_ELASTICSEARCH_HOSTS: "http://opensearch:9200"
    GRAYLOG_MONGODB_URI: "mongodb://mongo:27017/graylog"
  ports:
    - "9000:9000"    # Web UI / REST API
    - "1514:1514"    # Syslog TCP
    - "1514:1514/udp"
    - "5140:5140/udp" # Syslog UDP (rsyslog 전용)
    - "12201:12201/udp" # GELF UDP (Docker 컨테이너)
```

**접속 정보:**
- URL: `http://172.18.100.20:9000`
- 계정: `admin` / `clobot1q2w3er`

### 3-2. Input 설정 (API로 자동 생성)

| Input | 타입 | 포트 | 용도 |
|---|---|---|---|
| Syslog UDP | SyslogUDPInput | 5140 | 서버 syslog, auth.log |
| GELF UDP | GELFUDPInput | 12201 | Docker 컨테이너 로그 |

### 3-3. rsyslog 설정 (3개 서버 동일)

**파일:** `/etc/rsyslog.d/99-graylog.conf`

```
*.* @172.18.100.20:5140;RSYSLOG_SyslogProtocol23Format
```

적용 서버: pdkcld1, vm-server-01, gpu-server
```bash
sudo systemctl restart rsyslog
```

### 3-4. Docker 컨테이너 GELF 로깅

**대상 컨테이너:** gitlab (gitlab-srv), n8n, gitlab-runner, amr-monitor-web (vm-server-01)

docker-compose.yml 각 서비스에 추가:
```yaml

  options:
    gelf-address: "udp://172.18.100.20:12201"

```

### 3-5. Grafana OpenSearch 데이터소스 연동

- **URL:** `http://opensearch:9200` (내부 네트워크) 또는 `http://172.18.100.20:9200`
- **Index pattern:** `graylog_*`
- **Time field:** `timestamp`

## 4. Stream 설정 (보안/운영 알럿 자동 분류)

### 4-1. SSH 인증 실패 스트림

| 항목 | 내용 |
|---|---|
| 스트림명 | SSH Authentication Failures |
| 조건 | `message` contains `Failed password` OR `Invalid user` |
| 알럿 | 5분 내 10건 이상 → Jandi Webhook |

### 4-2. 서비스 크리티컬 에러 스트림

| 항목 | 내용 |
|---|---|
| 스트림명 | Service Critical Errors |
| 조건 | `level` <= 3 (emergency/alert/critical/error) |
| 알럿 | 5분 내 5건 이상 → Jandi Webhook |

### 4-3. HAProxy 5xx 에러 스트림

| 항목 | 내용 |
|---|---|
| 스트림명 | HAProxy 5xx Errors |
| 조건 | `source` = pdkcld1 AND `message` contains ` 5` (HAProxy log pattern) |
| 알럿 | 1분 내 10건 이상 → Jandi Webhook |

## 8. Graylog → Jandi 릴레이 구축 (2026-05-06)

### 배경
Graylog HTTP Notification은 자체 JSON 포맷을 전송하지만 Jandi Incoming Webhook은 별도 포맷 필요.
기존 jandi-adapter는 Prometheus Alertmanager 전용 → 변환 불가.

### 구성
- **스크립트:** `/usr/local/bin/graylog-jandi-relay.py`
- **포트:** `0.0.0.0:9876`
- **systemd:** `graylog-jandi-relay.service` (enable, active)
- **동작:** Graylog JSON → Jandi `{"body", "connectColor", "connectInfo"}` 변환 후 전달

### Graylog Notification 설정
- URL: `http://10.201.1.1:9876/` (Docker 게이트웨이 경유)
- URL 화이트리스트: `http://10.201.1.1:9876/` 등록

### 테스트 결과
- Graylog 컨테이너(10.201.1.4) → 릴레이 → Jandi: **HTTP 200 성공**
- journalctl: `10.201.1.4 - "POST / HTTP/1.1" 200`
