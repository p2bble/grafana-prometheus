- **작업자:** 심제범
- **대상 서버:** gitlab-srv (172.18.100.21), pdkcld1 (172.19.100.10), vm-server-01 (172.18.100.20)

## 1. 배경 및 문제 정의

### 기존 문제점
- gitlab-srv crontab에 `rsync -av --delete` 단순 명령만 등록되어 있었음
- NAS에는 GitLab 백업 tar 파일 2개(각 ~1.3TB)가 존재하나 로컬 백업 디스크는 1.8TB
- 기존 방식으로 rsync 실행 시 디스크 풀(No space left on device, exit 11) 발생
- rsync 성공/실패 여부를 Prometheus로 수집하지 않아 모니터링 불가
- node-exporter가 textfile collector 없이 실행 중 → .prom 파일 수집 불가
- Prometheus alert.rules.yml에 백업 관련 룰 없음

### 서버 구조
```
NAS (172.18.100.10)
  /volume1/source  →  gitlab-srv /source  (NFS 마운트)
  /volume1/storage →  gitlab-srv /storage (NFS 마운트)

백업 흐름:
  NAS /source/gitlab/current/backups/   (원본, 최대 2개 tar 파일 보관)
      ↓ 1차 rsync (03:00, gitlab-backup-rsync.sh)
  gitlab-srv /data/docker/gitlab/backups/  (로컬, 최신 1개만 유지)
      ↓ 2차 rsync (04:00, gitlab-secondary-rsync.sh)
  NAS /storage/backup/gitlab/secondary/  (2차 백업)
```

## 2. 작업 내용

### 2-1. node-exporter textfile collector 활성화 (gitlab-srv)

기존 node-exporter 컨테이너에 `--collector.textfile.directory` 옵션 누락 확인 후 재생성.

```bash
docker stop node-exporter && docker rm node-exporter

docker run -d \
  --name node-exporter \
  --restart always \
  --net host \
  --pid host \
  -v /:/host:ro,rslave \
  prom/node-exporter:latest \
  --path.rootfs=/host \
  --collector.textfile.directory=/host/var/lib/node_exporter/textfile
```

- textfile 경로: `/var/lib/node_exporter/textfile/` (호스트 기준)
- Prometheus scrape 경로: `gitlab-srv:9100` → 기존 설정 그대로 사용

### 2-2. 1차 rsync 스크립트 교체 (gitlab-backup-rsync.sh)

**파일 위치:** `/usr/local/bin/gitlab-backup-rsync.sh` (gitlab-srv)
**실행 권한:** root crontab `0 3 * * *`

**핵심 로직:**
1. NAS에서 가장 최신 .tar 파일 1개만 선택
2. 로컬에 이미 동일 파일이 있으면 skip (멱등성)
3. 다른 파일이 있으면 기존 .tar 삭제 후 rsync (디스크 풀 방지)
4. 성공/실패 결과를 Prometheus textfile로 기록

```bash
#!/bin/bash
set -uo pipefail

SRC_DIR="/source/gitlab/current/backups"
DST="/data/docker/gitlab/backups"
LOG="/var/log/gitlab-rsync.log"
PROM_DIR="/var/lib/node_exporter/textfile"
PROM_FILE="${PROM_DIR}/gitlab_rsync.prom"

mkdir -p "${PROM_DIR}"
echo "[$(date '+%Y-%m-%d %H:%M:%S')] rsync 시작" >> "${LOG}"

LATEST=$(ls -t ${SRC_DIR}/*.tar 2>/dev/null | head -1)
LATEST_FILE=$(basename "${LATEST}")

if [ -z "${LATEST_FILE}" ]; then
  echo "[$(date '+%Y-%m-%d %H:%M:%S')] 오류: 소스 백업 파일 없음" >> "${LOG}"
  exit 1
fi

echo "[$(date '+%Y-%m-%d %H:%M:%S')] 대상 파일: ${LATEST_FILE}" >> "${LOG}"

if [ -f "${DST}/${LATEST_FILE}" ]; then
  echo "[$(date '+%Y-%m-%d %H:%M:%S')] 이미 최신 상태 (${LATEST_FILE}), 건너뜀" >> "${LOG}"
  RESULT=1
else
  find "${DST}" -maxdepth 1 -name "*.tar" -delete
  echo "[$(date '+%Y-%m-%d %H:%M:%S')] 구버전 삭제 완료" >> "${LOG}"

  EXIT_CODE=0
  rsync -av "${LATEST}" "${DST}/" >> "${LOG}" 2>&1 || EXIT_CODE=$?

  if [ ${EXIT_CODE} -eq 0 ]; then
    RESULT=1
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] rsync 완료 (성공)" >> "${LOG}"
  else
    RESULT=0
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] rsync 실패 (exit: ${EXIT_CODE})" >> "${LOG}"
  fi
fi

printf '# HELP gitlab_rsync_success gitlab rsync 성공 여부\n# TYPE gitlab_rsync_success gauge\ngitlab_rsync_success %s\n# HELP gitlab_rsync_last_timestamp 마지막 rsync 완료 Unix timestamp\n# TYPE gitlab_rsync_last_timestamp gauge\ngitlab_rsync_last_timestamp %s\n' \
  "${RESULT}" "$(date +%s)" > "${PROM_FILE}"
```

### 2-3. 2차 rsync 스크립트 신규 생성 (gitlab-secondary-rsync.sh)

**파일 위치:** `/usr/local/bin/gitlab-secondary-rsync.sh` (gitlab-srv)
**실행 권한:** root crontab `0 4 * * *`

로컬 백업 디스크의 최신 파일을 NAS secondary 경로로 복사.

```bash
#!/bin/bash
set -uo pipefail

SRC="/data/docker/gitlab/backups/"
DST="/storage/backup/gitlab/secondary"
LOG="/var/log/gitlab-secondary-rsync.log"
PROM_DIR="/var/lib/node_exporter/textfile"
PROM_FILE="${PROM_DIR}/gitlab_secondary_rsync.prom"

mkdir -p "${DST}" "${PROM_DIR}"
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 2차 rsync 시작 (로컬 → NAS)" >> "${LOG}"

EXIT_CODE=0
rsync -av "${SRC}" "${DST}/" >> "${LOG}" 2>&1 || EXIT_CODE=$?

if [ ${EXIT_CODE} -eq 0 ]; then
  RESULT=1
  echo "[$(date '+%Y-%m-%d %H:%M:%S')] 2차 rsync 완료 (성공)" >> "${LOG}"
else
  RESULT=0
  echo "[$(date '+%Y-%m-%d %H:%M:%S')] 2차 rsync 실패 (exit: ${EXIT_CODE})" >> "${LOG}"
fi

printf '# HELP gitlab_secondary_rsync_success NAS 2차 rsync 성공 여부\n# TYPE gitlab_secondary_rsync_success gauge\ngitlab_secondary_rsync_success %s\n# HELP gitlab_secondary_rsync_last_timestamp 마지막 완료 Unix timestamp\n# TYPE gitlab_secondary_rsync_last_timestamp gauge\ngitlab_secondary_rsync_last_timestamp %s\n' \
  "${RESULT}" "$(date +%s)" > "${PROM_FILE}"
```

### 2-4. Crontab 최종 상태

**clobot user crontab** (`crontab -l`):
```
0 3 * * 0  /usr/local/bin/gitlab-weekly-backup >> /var/log/gitlab-backup.log 2>&1
```

**root crontab** (`sudo crontab -l`):
```
# 매일 02:00 전체 백업 + 7일 전 백업 삭제
0 2 * * 0 /usr/local/bin/gitlab-weekly-backup

# 매주 일요일 03:00시에 로그 정리
0 3 * * 0 find /var/log/gitlab -name "*.log" -mtime +7 -exec rm {} \;

# 1차 rsync: NAS → 로컬 (매일 03:00)
0 3 * * * /usr/local/bin/gitlab-backup-rsync.sh >> /var/log/gitlab-rsync.log 2>&1

# 2차 rsync: 로컬 → NAS secondary (매일 04:00)
0 4 * * * /usr/local/bin/gitlab-secondary-rsync.sh >> /var/log/gitlab-secondary-rsync.log 2>&1
```

> **root crontab으로 이전한 이유:** 백업 스크립트가 `/data/docker/gitlab/backups` (root 소유) 및 NFS 마운트 경로에 접근해야 하므로 root 권한 필요.

## 3. 트러블슈팅 기록

| 문제 | 원인 | 해결 |
|------|------|------|
| rsync exit 11 (No space left) | NAS에 1.3TB 파일 2개 → 로컬 1.8TB 디스크 풀 | 최신 파일 1개만 동기화하도록 스크립트 재설계 |
| set -e에서 rsync 실패 시 .prom 미생성 | set -e가 rsync 실패 즉시 스크립트 종료 | set -uo pipefail로 변경, EXIT_CODE 패턴 사용 |
| textfile 메트릭 미수집 (gitlab-srv) | node-exporter에 --collector.textfile.directory 옵션 누락 | 컨테이너 재생성으로 옵션 추가 |
| promtool check 경로 오류 | rules/ 디렉토리 마운트인데 직접 경로 사용 | /etc/prometheus/rules/alert.rules.yml로 수정 |
| curl reload 405 오류 | 포트 9090에 Cockpit 서비스 중복 | 실제 Prometheus 포트 9099 사용 |
| GitlabRsyncStale 알람 발화 | cron이 clobot 권한으로 실행 → 로그 파일(root 소유) 쓰기 실패 + NFS glob 미확장 | root crontab으로 이전, sudo bash -c → ls로 단순화 |
| sudo ls glob 미확장 | clobot이 /data/docker/gitlab/backups 접근 불가 → bash가 glob 확장 실패 후 리터럴 전달 | 스크립트를 root로 실행하여 sudo 불필요 |
| /data/docker/gitlab/backups Permission denied | Docker 볼륨이 root 소유 | root crontab으로 이전하여 해결 |
| vm-server-01 node-exporter 재시작 루프 | 기존 systemd node_exporter 서비스가 포트 9100 점유 → Docker 컨테이너 충돌 | systemd 서비스에 textfile 옵션 추가 후 재시작, Docker 컨테이너 제거 |
| python3 -c IndentationError | SSH 터미널 자동 들여쓰기 + 멀티라인 입력 | 항상 세미콜론으로 한 줄 작성 |
| vm-live-backup.sh running VM 디스크 복사 누락 | `qemu-img info`를 overlay(VM write lock 점유) 실행 → BACKING 빈 값 → 복사 skip | snapshot **전에** 원본 경로 기록, info 호출 제거. 검증: sec-spot 19G → 8.1G, 24분, VM 무중단 |
| croms-rx-UXUI volume 타입 디스크 누락 | awk 필터 `$4~/\.qcow2/`로 확장자 없는 volume 타입 skip | `collect_disks()` 함수로 file/volume 모두 처리, virsh vol-path로 실제 경로 변환 |
