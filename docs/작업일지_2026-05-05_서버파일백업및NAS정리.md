## 4. server-file-backup.sh 구축

### 스크립트 위치
- 로컬: `C:\Users\User\Desktop\infra\shell\server-file-backup.sh`
- 서버: `/usr/local/bin/server-file-backup.sh` (pdkcld1, vm-server-01, gpu-server 동일)

### 백업 대상
| 경로 | 내용 | 제외 항목 |
|---|---|---|
| `/etc/` | 시스템 설정 전체 | *.swp, *.bak |
| `/usr/local/bin/` | 운영 스크립트 | node_exporter, snmp_exporter, docker-compose, dropwatch, dwdump (바이너리) |
| `/data/docker/` | 컨테이너 구성 파일 | prometheus/data, grafana/data, loki/data, sonarqube-data, n8n, gitlab-runner/builds, gitlab-runner/cache, amr_monitor/build, mariadb, storage, *.iso, *.tar.gz |
| crontab 덤프 | root + clobot crontab | — |

### 백업 경로
```
/storage/backup/servers/<hostname>/
├── etc/
├── usr_local_bin/
├── data_docker/
├── root_crontab.txt
└── clobot_crontab.txt
```

### cron 등록 (3개 서버 동일)
```
0 5 * * * /usr/local/bin/server-file-backup.sh >> /var/log/server-file-backup.log 2>&1
```
매일 05:00 실행 (KVM 백업 01:00, SonarQube/n8n 02:00, Graylog 03:00, GitLab 03:00~04:00 완료 후)

### Prometheus 메트릭
- `server_file_backup_success{server="<hostname>"}` — 0/1
- `server_file_backup_last_timestamp{server="<hostname>"}` — Unix timestamp
- 파일: `/var/lib/node_exporter/textfile/server_file_backup.prom`
